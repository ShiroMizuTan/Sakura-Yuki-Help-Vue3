机厅人数统计 ​

功能简介 ​

该系统允许用户实时更新和查询机厅（如游戏中心、游乐场等）的当前人数。用户可以为店铺设置别名，并通过简单的指令进行人数更新和查询。

INFO

系统状态：已全面开放使用

指令说明 ​

店铺查询与设置 ​

指令格式说明
查找店铺IDfind_id [微信公众号处店铺名称]返回对应店铺的ID，用于后续设置
设置店铺别名set_arcade [别名] [店铺id]为店铺设置一个简短别名（仅限本群使用）
查看别名列表list_aliases列出本群设置的所有店铺别名及其ID
删除别名del_alias [别名id]删除本群设置的特定别名

人数更新 ​

指令格式说明
设置人数[别名]=[数字] 或 [别名] [数字]将指定店铺人数更新为指定数字
增加人数[别名]+[数字]在最近12小时内更新的人数基础上增加指定数量
减少人数[别名]-[数字]在最近12小时内更新的人数基础上减少指定数量

人数查询 ​

指令格式说明
查询机厅人数几卡 或 jk显示最近4小时内本群所有别名的店铺人数数据

使用示例 ​

基础设置流程 ​

1. 查找店铺ID：
   find_id 上海环球港世宇乐园
   → 返回：店铺ID为 12345

2. 设置别名：
   set_arcade 环球港 12345
   → 成功设置别名

3. 查看已设置别名：
   list_aliases
   → 1. 环球港 (ID: 1)

人数更新示例 ​

1. 设置当前人数：
   环球港=15
   → 环球港：15人

2. 增加人数：
   环球港+3
   → 环球港：18人（在最近12小时记录的基础上+3）

3. 减少人数：
   环球港-5
   → 环球港：13人（在最近12小时记录的基础上-5）

查询示例 ​

输入：几卡
输出：
最近4小时机厅人数：
1. 环球港：13人（更新时间：10:30）
2. 中山公园店：7人（更新时间：11:15）

INFO

此处只做简单演示，与实际返回内容不一致

注意事项 ​

WARNING

重要规则说明

  1. 人数更新基于最近12小时内的记录进行加减
  2. 几卡/jk 指令仅显示最近4小时内有更新的店铺
  3. 别名设置仅对本群有效，不同群可设置不同的别名
  4. 店铺ID需通过 find_id 指令获取，确保准确性

TIP

使用建议

  • 建议使用易记的短别名，如地点缩写
  • 定期使用 list_aliases 清理不再需要的别名
  • 人数更新后，建议等待几分钟再查询，确保数据同步

数据更新逻辑 ​

原始数据 → [别名]=[数字] → 设置绝对数值
          [别名]+[数字] → 在最近12小时记录上增加
          [别名]-[数字] → 在最近12小时记录上减少

查询时 → 仅显示最近4小时内有更新的数据

通过此系统，用户可以方便地共享和查询机厅实时人数，合理安排出行时间，避免拥挤。
